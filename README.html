README¶

独立进行软件的开发具有较高的灵活性、较为方便，但难以规避程序设计过程中主观策略带来的偏性以及开发测试过程中留存的错误。引入Code Review机制，以期再一定程度上规避所述问题，同时在未来新增的流程内容中实现”一份代码多人了解”避免人员变动交接而引发的流程维护缺失。

本文档旨在基于git平台建立健全生信流程开发过程中的有效代码评审机制，下文测试基于Genomics Gitlab 实现，其在github 等类似工程管理平台亦有相同的实现。

问题与意见反馈: liubo4@genomics.cn / https://gitlab.genomics.cn/Tumor_BioInfo/codereview/-/issues

参与文档撰写：https://gitlab.genomics.cn/Tumor_BioInfo/codereview
